# ISO/IEC TR 24027:2021 — Bias in AI Systems and AI-Aided Decision Making

## Scope
This technical report provides guidance on bias in AI systems and AI-aided decision making. It addresses sources of bias, mitigation techniques, and evaluation methods.

## Sources of Bias
Bias can arise from training data (representation, measurement, aggregation), algorithmic design (optimization objectives, feature selection), and deployment context (human-in-the-loop decisions, feedback loops).

## Mitigation Techniques
Pre-processing methods include data augmentation, resampling, and reweighting. In-processing methods include adversarial debiasing and fairness constraints. Post-processing methods include threshold optimization and calibrated equalized odds.

## Evaluation
Bias should be evaluated using multiple fairness metrics (demographic parity, equalized odds, calibration) and intersectional analysis across multiple protected attributes.
